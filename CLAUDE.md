# Crown Core — CLAUDE.md

Этот репозиторий содержит **только ядро**. Ядро — два артефакта и больше ничего:

1. **`splitter`** — неизменяемая донат-программа на Solana. Внутри транзакции донора переводит донат **целиком** получателю и эмитит `Settled`. Комиссии не берёт, баланса не имеет.
2. **`crown-index`** — канистра ICP. Читает `Settled` от пиннутого сплиттера, сворачивает в книгу репутации, отдаёт её на чтение. Денег нет, ключей нет, подписи нет; единственный update — пустой будильник `ingest_hint`, влияющий только на момент следующего чтения (core-spec §5).

Всё остальное — игры, эскроу, фабрика, фронт, бэкенд, модерация, косметика — **не здесь**. См. `docs/post-core.md`.

Этот файл читается каждую сессию. Держи коротким.

## Незыблемые принципы

1. **Ядро только считает.** `book = fold(reduce, empty, settlements)`. Чистая, детерминированная, версионированная функция без I/O.
2. **Репутация ≡ честно уплаченный доллар.** 1 USD gross, дошедший до получателя через сплиттер = 1 единица репутации. Линейно, без decay, монотонно, непередаваемо, per-wallet, локально для получателя. Дельт нет. Дебета нет. Возврата нет.
3. **Единственный вход в книгу — пиннутый сплиттер.** `Settled` от любой другой программы игнорируется. Не «любая программа с событием `Settled`».
4. **Сплиттер не держит денег и не берёт комиссий.** Средства идут донор → получатель одним прямым переводом внутри транзакции донора. Программа не является промежуточным владельцем ни на один слот. Ядро бесплатно с обеих сторон: эмиссия репутации и чтение книги ничего не стоят.
5. **Индексатор не знает про комиссии.** Монетизация — комиссии игр: они снимаются эскроу **до** сплиттера и в книгу не попадают. Индексатор доверяет **адресу**, а не числам в событии. Честность чисел обеспечена неизменяемостью программы.
6. **Границы — компилятором, не договорённостью.** `crown-reduce` — крейт с нулём зависимостей. Он физически не может сделать I/O. Это проверяется в CI, а не в ревью.
7. **Чтение чейна — только через RPC-канистру под NNS.** `SOL RPC` (`tghme-zyaaa-aaaar-qarca-cai`), с мульти-провайдерным консенсусом. На mainnet — только `Default`-источники. Свои URL и API-ключи запрещены.
8. **Минимализм = безопасность.** У каждой строки есть текущий потребитель. Никакой обобщённости «на будущее». Сомневаешься между «добавить» и «удалить» — удаляй.

Расхождение кода с принципами или с `docs/` = баг. Не «решаем на месте»: правь код; если неверен принцип — сначала правь доку.

## Структура репозитория

```
crown-core/
├─ CLAUDE.md
├─ docs/
├─ reduce/                 # КРЕЙТ БЕЗ ЗАВИСИМОСТЕЙ. Чистая свёртка.
│  └─ src/{lib,event,book,reduce}.rs
├─ index/                  # КАНИСТРА. Только query в Candid.
│  └─ src/{lib,api,cursor,certify,source/{mod,solana}}.rs
├─ contracts/
│  └─ solana/              # Anchor. Upgrade authority сожжён.
└─ config/
   ├─ testnet.toml
   └─ mainnet.toml
```

Зависимости строго в одну сторону: `index → reduce`. Обратной нет. `reduce` не знает про ICP, про Solana, про Candid.

## Как работать

1. Прочитай `docs/build-plan.md`, определи текущий этап (первый не-`DONE`).
2. Работай строго по нему. Не забегай вперёд.
3. Этап не закрыт, пока Definition of Done не выполнен и тесты зелёные.
4. По завершении — обнови статус, удали устаревшую доку тем же коммитом, только потом следующий этап.

## Гардрейлы (NEVER)

- Никогда не добавляй в `reduce` зависимость. Ни одну. `serde`, `candid`, `ic-cdk` — в `index`, конверсия на границе.
- Никогда не экспортируй `update`-метод сверх пустого будильника `ingest_hint`. Будильник не несёт ни аргументов, ни ответа: влиять на содержимое книги вызовом невозможно, только приближать момент чтения, и не чаще зазора. Всё остальное в `.did` — только `query`.
- Никогда не засчитывай `Settled` от адреса, отличного от пиннутого сплиттера.
- Никогда не давай ядру держать деньги, ключи, подписывать, резолвить исходы.
- Никогда не вводи дельты репутации, дебет, сгорание, decay.
- Никогда не пиши сплиттер так, чтобы он в какой-то момент владел токенами. Ноль баланса — инвариант, а не следствие.
- Никогда не добавляй в сплиттер комиссию, казну или второй перевод. Монетизация — комиссии игр вне ядра (`docs/post-core.md`); ядро денег не зарабатывает.
- Никогда не оставляй у программы сплиттера upgrade authority после финального деплоя и не добавляй admin-инструкций.
- Никогда не используй `Custom` RPC-источники в mainnet-профиле.
- Никогда не хардкодь сетевые значения. Только `config/`.
- Никогда не используй `unwrap`, `expect`, `panic!`, немаркированную арифметику в `reduce`.
- Никогда не добавляй код «на будущее» без текущего потребителя. Если хочется — это `docs/post-core.md`.

## Стек

- `reduce`: Rust, `#![no_std]`-совместимый по духу, zero deps, `#![forbid(unsafe_code)]`.
- `index`: Rust, `ic-cdk`, `ic-stable-structures`, клиент SOL RPC-канистры.
- `contracts/solana`: Anchor.
- Доки и проза — по-русски. Идентификаторы, комментарии, имена файлов кода — по-английски.
