# Crown Core — Стандарты

Правило отбора: **в этом файле только то, что можно проверить машиной или тестом.** Всё, что проверяется вкусом, — не стандарт, а мнение, и здесь ему не место.

## Границы (проверяются CI, не ревью)

| Граница | Как заявлена | Как проверена |
|---|---|---|
| `reduce` не делает I/O | ноль зависимостей | `cargo tree -p crown-reduce` печатает один крейт |
| `reduce` не знает про ICP/чейны | ноль зависимостей | `grep -rE 'ic_cdk\|std::(fs\|net\|time)' reduce/src/` пусто |
| Никто не пишет в канистру | единственный не-`query` — пустой будильник `ingest_hint` без аргументов | парсер `.did` падает на любой другой не-`query` метод |
| Индексатор не доверяет одиночному RPC | только `Default`-источники | линт mainnet-профиля на `Custom` |
| Индексатор не знает про комиссии | в конфиге нет чьих-либо комиссий и казны | схема конфига их не содержит |
| Сплиттер не владеет средствами | один прямой перевод | тест «token account не существует» |

Зависимость строго `index → reduce`. Обратной нет и быть не может: `reduce` о существовании `index` не знает.

## Код

**`reduce`:**
```rust
#![forbid(unsafe_code)]
#![deny(clippy::unwrap_used, clippy::expect_used, clippy::panic,
        clippy::arithmetic_side_effects, clippy::indexing_slicing)]
```
Только `checked_*` арифметика. Исчерпывающие `match`, никаких `_ =>` на доменных enum. Ошибка — значение, не паника.

**`index`:** `clippy` без предупреждений. `unwrap`/`expect` запрещены на пути ингеста; в `#[init]` допустимы, там паника означает несостоявшийся деплой.

**Программа:** максимально мало логики. Anchor, `transfer_checked`, никаких `unchecked` аккаунтов.

**Общее.** У каждой строки есть текущий потребитель. Ни мёртвого кода, ни обобщённости «на будущее», ни фича-флагов. Сомневаешься между «добавить» и «удалить» — удаляй.

## Тесты, без которых этап не закрывается

1. Закон репутации: монотонность, аддитивность, пересчёт, изоляция ключа, детерминизм.
2. Гарантии сплиттера: `out == in` (получатель получает ровно `gross`), нулевой донат ревертится, донор структурен, ноль баланса.
3. Признание: `Settled` не от пиннутого адреса не засчитан.
4. Exactly-once: повторный ингест с того же курсора не меняет книгу.
5. Перекрёстная сверка события с исполненными переводами.
6. Иммутабельность: upgrade authority сожжён.
7. Сертификат query проверяется против root key NNS.

Тесты лежат рядом с исходником, который проверяют.

## Конфиг

Единая `config/`. Ничего сетевого в коде.

Два профиля: `testnet`, `mainnet`. Смена среды = смена профиля, ноль изменений кода.

Per-chain ровно: `id`, `source`, `consensus`, `splitter`, `usdc`, `factories` (список, ноль или больше). Всё.

Нет и не появится: RPC-URL, API-ключ, чьи-либо комиссии, адрес казны, `min-donation`, `finality_depth`, схема подписи. Первые два запрещены; комиссии — параметры эскроу игр (`crown-factory`) и ядру неизвестны; единственный параметр деплоя контракта — минт USDC в `contracts/*/deploy.toml`.

Только нативный USDC (Circle). Bridged-обёртки запрещены; вход из других сетей — CCTP-пополнение кошелька на уровне crown-app (mint нативного USDC).

## Циклы

Ядро на внешних сетях газ **не платит вообще** и ключей для этого не держит.

- **Газ за донат — донор.** Его транзакция, его SOL. Центы.
- **Циклы канистры — оператор.** Reverse gas: читатель книги не платит никогда. Расход: outcalls ингеста (× число провайдеров × число нод подсети) + permissionless-query. Источник финансирования — вне ядра (у ядра нет казны); механизм пополнения — S4.
- Permissionless-query это вектор DoS. Принято как ограничение (core-spec §10.4), митигация — бюджет и rate limiting на boundary nodes, не код ядра.

## Именование

- Файлы кода: kebab-case. Типы: PascalCase. Функции/переменные: snake_case. Константы: UPPER_SNAKE_CASE.
- Идентификаторы и комментарии — по-английски. Доки и проза — по-русски.
