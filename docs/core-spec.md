# Crown — Спецификация ядра

**v4.0 · Сплиттер + индексатор · Больше ничего**

Заменяет v3.0 целиком. Отличия: убрана фабрика (ядру не нужна), убран отдельный verifier (растворился в чистом крейте `reduce`), убраны committed-эскроу, игры, централизованный слой, фронт — всё в `post-core.md`. Признание перестало быть выводимым из событий и стало пиннутым адресом.

---

## 1. Ядро — это два артефакта

| Артефакт | Где | Держит деньги | Держит ключи | Изменяем |
|---|---|---|---|---|
| `splitter` | внешняя сеть | **нет, никогда** | нет | нет |
| `crown-index` | канистра ICP | нет | нет | нет (после S5) |

Между ними нет канала. Сплиттер не знает об индексаторе. Индексатор читает сплиттер из публичного чейна и больше ничего от него не хочет.

```
донор ──транзакция──► splitter ──┬──► стример   (gross − fee)
                                 └──► казна     (fee)
                                 
                             emit Settled(payer, streamer, gross, fee)
                                        │
                     RPC-канистра (NNS) │ читает, консенсус ≥3 провайдеров
                                        ▼
                                  crown-index
                                  reduce → книга
                                        │
                                  query │ бесплатно, permissionless, certified
                                        ▼
                                  кто угодно
```

## 2. Закон репутации

Книга — отображение `(chain, payer, streamer) → gross`, где `gross` — сумма USDC в minor units (6 знаков), `u128`.

```
reduce(book, Settled { chain, payer, streamer, gross, .. })
    = book.insert((chain, payer, streamer), book.get(...) .checked_add(gross)?)
```

Всё. Одно правило, одна ветка.

**Свойства, которые из него следуют и обязаны быть доказаны тестами:**

- **Монотонность.** Значение только растёт. Нет операции, уменьшающей баланс.
- **Аддитивность / нечувствительность к порядку.** `fold` над любой перестановкой множества сеттлментов даёт ту же книгу. Кросс-чейн упорядочивание не нужно и не существует.
- **Идемпотентность на уровне ингеста.** Один сеттлмент не начисляется дважды. Обеспечивается курсором (§5), не памятью о виденном.
- **Пересчёт.** `book[(c,p,s)] == Σ gross` по всем `Settled` этой тройки. Тривиально проверяемо.

**Начисляется плательщику**, то есть тому, с чьего кошелька реально утянули токены. Не тому, кто вызвал. Это обеспечено контрактом (§3), а не параметром события.

**Ключ включает `chain`.** Кошелёк на Solana и кошелёк на Base — разные субъекты. Ядро не связывает адреса между сетями и не имеет для этого механизма. Связывание личностей — задача отображения, оно никогда не питает книгу (см. `post-core.md`).

**Возврата нет, дельт нет, decay нет.** Репутация тождественно равна сумме честно уплаченного. Она непередаваема, непродаваема, локальна для стримера. Потеря ключей = потеря репутации; это свойство непередаваемости, не баг.

## 3. Сплиттер: три гарантии, ноль баланса

Один immutable контракт на сеть. Не фабрика, не шаблон, не инстанс — **один адрес**.

Единственная функция: `donate(streamer, gross)`.

```
fee    = gross * FEE_BPS / 10000          // floor
payout = gross - fee                      // out == in, точно
transferFrom(msg.sender → streamer, payout)
transferFrom(msg.sender → TREASURY, fee)
emit Settled(payer = msg.sender, streamer, gross, fee)
```

**Гарантия 1 — `gross` = реально сдвинутые деньги.** Не декларация. Если перевод не прошёл, транзакция ревертится, события нет.

**Гарантия 2 — плательщик = реальный фандер.** `payer` не параметр. Токены тянутся с `msg.sender` (EVM) / с ATA, чей authority подписал транзакцию (Solana). Подарить репутацию чужому адресу невозможно, даже за реальные деньги.

**Гарантия 3 — контракт никогда не имеет баланса.** Два прямых перевода донор→получатель. Контракт не является владельцем средств ни в один момент. Это не «он не крадёт» — **красть нечего**. Non-custodial по построению, а не по логике.

**Комиссионный флор.** `require(fee > 0)`. При `FEE_BPS = 300` и 6 знаках это `gross ≥ 34` minor units. Иначе микро-донат эмитил бы репутацию бесплатно.

**Неизменяемость — часть спеки, не операционка.**
- EVM: нет `owner`, нет `admin`, нет proxy, нет `delegatecall`, нет `selfdestruct`. `FEE_BPS` и `TREASURY` — `immutable`, заданы в конструкторе.
- Solana: после деплоя `solana program set-upgrade-authority --final`. DoD проверяет, что `solana program show` печатает `Authority: none`.

Контракт крошечный (цель: < 100 строк логики) именно потому, что он единственное место, где может быть настоящая дыра.

## 4. Признание: пиннутый адрес

Индексатор засчитывает `Settled` **только от адреса сплиттера этой сети, взятого из `config/`**. Больше ни от кого.

Почему не «любой контракт с `Settled`»: злоумышленник эмитит `gross = 1e18` со своего контракта → печатает репутацию даром.
Почему не «прислал 3% в казну»: платит 3% с копейки, эмитит огромный `gross` → та же дыра.
Почему не фабрика: фабрика была нужна, чтобы порождать эскроу-инстансы игр. Игр в ядре нет. Корень доверия у фабрики был **тот же самый** — её адрес per-chain в конфиге. Значит фабрика добавляла один уровень косвенности и ноль безопасности.

Корень доверия = `(адрес сплиттера в конфиге) × (неизменяемость сплиттера) × (неизменяемость индексатора)`. Три множителя, каждый проверяем публично.

## 5. Индексатор: ингест, курсор, exactly-once

Ингест — внутренний таймер. **Публичных `update`-методов нет.** Никакой вызывающий не может повлиять на состояние. Это самое сильное разграничение, доступное на ICP, и оно бесплатное.

**Финальность решается коммитментом, не константой.** В конфиге нет `finality_depth`.
- Solana: `commitment = finalized`.
- EVM: `BlockTag::Finalized`.

Реорг до финальности → события просто нет в свёртке.

**Курсор — единственное состояние ингеста.**

- **EVM:** `eth_getLogs { address: splitter, topics: [Settled], fromBlock: cursor + 1, toBlock: Finalized }`. Курсор = номер последнего обработанного блока. Одна outcall на пачку.
- **Solana:** `getSignaturesForAddress(program_id, { until: last_sig, commitment: finalized })`, пагинация через `before`, обработка от старых к новым; затем `getTransaction(sig)` на каждую новую подпись. Курсор = последняя финализированная подпись.

Курсор монотонен + финальность необратима ⇒ exactly-once без множества «виденных». Память O(1).

**Solana: не полагайся на `logMessages`.** Программные логи усекаются при переполнении. Событие писать через event-CPI (оно попадает в inner instructions и не усекается).

**Факт (проверено на S3, devnet):** SOL RPC при `encoding = base64` отдаёт структурированную мету целиком — `meta.innerInstructions` (со `stackHeight`), `meta.pre/postTokenBalances`, `meta.loadedAddresses`. Событие распознаётся как inner instruction пиннутой программы с тегом event-CPI; подделать его нельзя — инструкция требует подписи event-authority PDA, которую даёт только сама программа.

**Перекрёстная сверка — по исполненным переводам, не по дельтам балансов.** Дельты `pre/postTokenBalances` неоднозначны при совпадении ролей (payer == streamer, streamer == treasury), поэтому сверяются сами исполненные инструкции: два `TransferChecked` непосредственно перед событием на той же глубине стека обязаны давать `payout + fee == gross`, `fee == event.fee > 0`, минт — USDC из конфига, authority обоих — `payer` из события, источник общий. Расходятся — сеттлмент не засчитывается, инкремент счётчика аномалий. Два независимых свидетельства — событие и переводы — должны совпасть.

## 6. Открытая книга

Все методы канистры — `query`. Бесплатно, permissionless. Читать может кто угодно, строить поверх может кто угодно; его чтение на книгу не влияет.

```
get_reputation(chain, payer, streamer) -> u128
get_cursor(chain)                      -> Cursor
get_reduce_version()                   -> u32
get_certificate()                      -> (Certificate, BookRoot)
```

**Certified data.** Обычный `query`-ответ подписан одной репликой и потому третьему лицу не доказывает ничего. Индексатор держит меркл-корень книги в `set_certified_data`, а `query` отдаёт `data_certificate()`. Ответ проверяется против root key NNS без доверия оператору.

Именно это заменяет отдельный проект-verifier. А кому мало сертификата — берёт крейт `reduce` (нулевые зависимости, публичный, версионированный), скармливает ему те же публичные `Settled` и получает ту же книгу. «Пересчитай сам» — это `cargo run`, а не репозиторий.

## 7. Конфиг-поверхность

Индексатор знает про сеть ровно четыре вещи:

```toml
[[chain]]
id       = "solana-devnet"        # внутренний ChainId ядра
source   = "Default:Devnet"       # вариант RPC-канистры; на mainnet только Default:*
consensus = "3-of-5"              # стратегия агрегации провайдеров
splitter = "<program id>"         # корень признания
usdc     = "<mint / erc20 address>"  # для перекрёстной сверки дельт
```

**Чего в конфиге индексатора НЕТ:** `feeBps`, адрес казны, `min-donation`, RPC-URL, API-ключи, глубина финальности, схема подписи.

`feeBps`, казна и флор — параметры **деплоя контракта** (`contracts/*/deploy.toml`). Индексатор их не читает и не проверяет. Он не может быть обманут неверным значением комиссии, потому что не имеет мнения о комиссии. Он доверяет адресу.

testnet → mainnet и новая сеть = смена профиля. Ноль изменений кода.

## 8. Целевые сети

Ограничение жёсткое: **только `Default`-источники RPC-канистр под NNS.** Всё, что требует своего URL (`Custom`), на mainnet запрещено — иначе оператор становится единственным источником истины и вся конструкция бессмысленна.

| Сеть | RPC-канистра | Default на mainnet | Default на тестнете |
|---|---|---|---|
| Solana | SOL RPC | ✅ Mainnet | ✅ **Devnet** |
| Ethereum | EVM RPC | ✅ | ✅ Sepolia |
| Base | EVM RPC | ✅ | ❌ только Custom |
| Arbitrum One | EVM RPC | ✅ | ❌ только Custom |
| Optimism | EVM RPC | ✅ | ❌ только Custom |

Следствия, которые определяют план сборки:

- **Solana идёт первой.** Единственная сеть, у которой и mainnet, и тестнет покрыты дефолтными провайдерами. Плюс девнет дешёвый и с фаусетом.
- **EVM тестируется на Ethereum Sepolia**, а отгружается на Base / Arbitrum / Optimism. Ethereum L1 поддержан, но как цель для донатов дорог.
- **Polygon, Avalanche и всё прочее — вне.** Они достижимы только через `Custom` = свои эндпоинты = запрещено. Не «пока не добавили», а **исключены требованием**.
- `Custom` разрешён **только в testnet-профиле**. Линт в CI: mainnet-профиль, содержащий `Custom`, роняет сборку.

RPC-канистры сами обращаются к третьесторонним провайдерам (Alchemy, Ankr, dRPC, Helius, Chainstack, PublicNode) и агрегируют ≥3 ответов. «Без посторонних RPC» невозможно; достижимо «без **своих** RPC и своих ключей, с консенсусом под управлением NNS». Это и есть требование.

## 9. Неизменяемость

Сплиттер immutable с момента деплоя — иначе гарантии §3 ничего не стоят.

Индексатор замораживается (blackhole) на S5. Обоснование не «чтобы не украли» — красть нечего, — а **честность закона**: замороженный `reduce` нельзя переписать задним числом. Это единственная гарантия, потому что независимого verifier-сервиса у нас нет.

Цена честная: **баг в замороженном индексаторе неисправим.** Митигация не в апгрейде, а в природе книги: она derived. Битый индексатор выбрасывается и заменяется новым, который пересчитывает всё из чейна с нуля. Активом является пара (адрес сплиттера, история чейна), а не канистра. Поэтому заморозка дёшева.

Требование до заморозки: **reproducible build**, хэш wasm опубликован. Иначе «заморожен» ничего не значит.

## 10. Принятые ограничения

Не латать, а принять.

1. **Самосибил возможен.** Кошельки не связываются, можно донатить себе с многих. Деньги возвращаются себе же, нам — 3% с каждого прохода. На уровне примитива это не атака: осмысленный вес стоит реальных денег.
2. **Репутация — плутократия по построению.** В money-backed системе это свойство. Разбавить «заработанной» компонентой ядро не может — репутация тождественна деньгам.
3. **Репутация чейн-локальна.** Один человек с кошельками в двух сетях имеет две репутации у одного стримера. Склейка — вне ядра и никогда не пишет в книгу.
4. **Permissionless-чтение жжёт циклы.** Query бесплатны для читателя и оплачиваются канистрой (reverse gas). Это статья расхода и вектор DoS. Бюджет — на S5.
5. **Доверие к RPC-канистрам.** Мы доверяем NNS-управляемому коду и его наборам провайдеров. Это сокращение доверия, а не его отсутствие.
6. **Ядро требует дешёвой сети.** Донат должен стоить центы.

## 11. Монетизация ядра

- 3% с каждого сеттлмента через сплиттер. Это плата за **эмиссию** репутации, а не за исход.
- Чтение книги бесплатно и permissionless. Кран платный только на запись.
- Комиссия приходит в USDC на внешних сетях. Конвертация USDC → cycles в казну канистры — операционка, S5.

Репутация читается всеми даром; начислить её можно только заплатив. Это и граница безопасности, и бизнес-модель — одна и та же граница.
